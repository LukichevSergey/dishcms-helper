ВАЖНО!
Сайт должен быть установлен в DOCUMENT_ROOT (корневая папка домена, либо поддомена вида subdomain.example.com)
Из вложенной папки сайт работать не будет (http::/example.com/mysite/)

Инсталляция:
Скопировать файлы из папки public_html в папку сайта (DOCUMENT_ROOT) 
Скопировать папку yii в папку на один уровень выше папки сайта.
Примечание: 
Yii Framework можно подключить из любой папки. 
В этом случае, необходимо вручную установить путь к файлу yiilite.php в файле index.php
$yii='мойпутькфрэймворку/yiilite.php';

Прописать настройки соедниения с базой данных в файле \protected\config\db.php:
Заменить соответственно: "имябазыданных", "имяпользоваля", "пароль".

Дапм базы данных накатится при первом обращении к сайту (необходимо подождать загрузки сайта). 
Браузер может выдать сообщение о том, что сайт недоступен, перезагрузите страницу, сайт должен заработать.

----------------------
Требования к хостингу:
----------------------
Apache
5.4+ версия PHP,
MySQL 5.x

-----------------------------
Требования к PHP расширениям:
-----------------------------
PDO MYSQL - библиотека для работы с базой данных MySQL.
CURL - для авторизации через наш сервер
GD - GD для изображений обработки
EXIF - библиотека для работы с мета-данными изображений.
MBSTRING - библиотека для работы с многобайтовыми строками (UTF-8).


Если на сервере установлен ImageMagick, желательно переключится на его использование:
\protected\config\defaults.php:
	...
	'image' => array(
    	'class'=>'ext.image.CImageComponent',
        // GD or ImageMagick
        'driver'=>'GD',
        ...
заменить соответственно GD на ImageMagick.
	'driver'=>'ImageMagick',

-----------------------------------------
Возможные ошибки (если сайт не заработал)
-----------------------------------------
Проверьте права доступа
На папки должны быть разрешены права на запись от имени пользователя под которым работает веб-сервер Apache.
DOCUMENT_ROOT/assets/
DOCUMENT_ROOT/assets/css
DOCUMENT_ROOT/protected/runtime/
DOCUMENT_ROOT/images/
